Getting Started

Welcome

We’re really excited to have you using CloudCannon. This guide will take you from knowing absolutely nothing about CloudCannon to having an online website your client’s can update... and it only takes 5 minutes!

Initial Setup

First up we need a CloudCannon account. Head over to the CloudCannon homepage and click on the Try it Free button.

Now you need to give CloudCannon access to your Dropbox account. Click Connect to Dropbox.

Note that CloudCannon can only see files inside the special CloudCannon folder Dropbox creates when you allow access, the rest of your files remain private

Login with your Dropbox account.

Click Allow to give CloudCannon access to the new "CloudCannon" folder in your Apps folder.

Now enter a password for your CloudCannon account and click the Sign up.

Creating your first site

You should be signed up now. Let’s create our first site! Click the Add New Site button.

First, enter a site name which is just a friendly name we use when referencing your site in CloudCannon. Now we need to choose a domain. Here you have two choices: you can either use an address like yoursite.cloudvent.net or use your own registered domain. In this tutorial we’ll enter a cloudvent address as it’s easier to get going but you can always switch later on. Press Create Site when you’re finished.

Your website is now created! CloudCannon will add a folder to your Dropbox account for your new site. Click Open your site.

Lets add some files to your site. You can either use your own website or download a template to get going. I’m going to download a template called Blanka from freebiesbooth.com. Copy your website files into your site folder Dropbox/Apps/CloudCannon/your-domain-name.com.

Your website is now live on the internet! Go to your domain and check it out.

Giving your client access

Now we need to give a client access to update their website. Head back to CloudCannon, open up your site and click on the index.html file. This will open the page up in code view. Let’s say we want the client to be able to update the introduction section. Scroll down to line 58 where you find the introduction section in the html. The div already has a class of section , all we need to do is add editable so it becomes <div class="section about editable">. Press Save Changes in the bottom right corner.

Let’s make sure that worked. Click on Switch to Editor in the bottom left corner. The web page will come up, when you move the mouse a yellow box appears around the introduction section. Try clicking the section and updating the content.

Next we need to share the site with the client. Click the back arrow at the top left to exit edit mode. Then click on the sharing bottom at the bottom.

Enter a password your clients will enter when they update their website. Then press Update Password.

Go to yourwebsite.com/update, type in the password you entered before and you’ll be logged in to the client editor. Click on the index file, make an edit to the introduction section, hit save changes then go to yourwebsite.com and see the changes.

All you need to do now is email your client the url and password they need to go to update their site.

That’s all there is to it! Get in touch anytime you need help setting your website up, reach us at support@cloudcannon.com.

Full Documentation

Dropbox

CloudCannon uses the power of Dropbox to sync changes from your computer to our servers and push changes your client makes on our servers back to your computer.

CloudCannon can only see files inside a special CloudCannon folder Dropbox creates when you allow CloudCannon access to your Dropbox, the rest of your files remain private.

There's a 200MB limit on files synced through Dropbox or the website. Anything bigger probably belongs on YouTube or a 3rd party file sharing website.

Editable Regions

Making your website updatable takes a few seconds on CloudCannon. Open up your html and add a class of editable to any elements you want your client to update. That’s it! See Sharing Websites to give your client access to update.

At the moment we support adding an editable class to the following elements:

  • Block elements: div, section, article, aside, footer, header, nav
  • Text elements: p, h1, h2, h3, h4, h5, h6
  • Special elements: img

Repeatable Regions

With repeatable regions you can add a single class to your HTML to give your clients the ability to add, move and delete a repeating element. Repeatable regions are useful in a number of situations such as updating photos in a photo gallery, adding a news section or updating staff profiles.

All you need to do is add a class of repeatables to an element in your HTML. The element's direct children then become repeatable regions which can be updated by the client.

I'll give a quick run through of how it works using a news section as an example. First, add the HTML for a single news item and then wrap it in a div with a class of repeatables: 

<div class="repeatables">
	<div class="editable">
		<h2>10/01/13 - Exciting News Item!</h2>
		<p>Text for the news item.</p>
	</div>
</div>

Save this and switch to the editor view. The news item now has a yellow icon at top left.

Click on the icon and the news item will be cloned. The HTML will now look like this:

<div class="repeatables">
	<div class="editable">
		<h2>10/01/13 - Exciting News Item!</h2>
		<p>Text for the news item.</p>
	</div>
	<div class="editable">
		<h2>10/01/13 - Exciting News Item!</h2>
		<p>Text for the news item.</p>
	</div>
</div>

The editable class works with repeatable regions so the content can be updated using the normal editor. Hovering over the news items now displays arrow icons for reordering and a trash can to delete a news item.

Custom Domain Name

These instructions are for those who have already purchased a custom domain name. If you have not yet purchased a custom domain, go to a 3rd party domain provider (iwantmyname automatically configures itself with CloudCannon) and purchase a domain.

What is DNS?

The Domain Name System (DNS) translates Internet domain and host names to IP addresses. DNS automatically converts the names we type in our Web browser address bar to the IP addresses of Web servers hosting those sites.

There are two ways of setting up your own domain on CloudCannon, either you can use your domain registrar's DNS server or you can use CloudCannon's. Making changes to your DNS can take up to 48 hours to propagate around the Internet.

Use your Domain Registrar's DNS server

Most domain registrars offer a DNS server, if your provider doesn't then try using CloudCannon's DNS server.

  1. Go into the DNS settings for your domain on your domain registrar's website.
  2. Add a new record.
  3. Enter the following details:

    Name/Alias - leave blank to configure the root domain (e.g. mysite.com) or enter a value for a subdomain (e.g. enter www to point www.mysite.com at CloudCannon).

    Type - Set to A.

    Value - Set to 184.169.135.34.

One last thing to note: It's a common situation in web hosting to redirect subdomains to the main domain. For example, let's say you have www.mysite.com and you want visitors who go to mysite.com to be redirected to www.mysite.com. Setting this up on CloudCannon is easy, just create DNS records for each domain you want to redirect to your main domain and CloudCannon will handle the redirects.

Use CloudCannon's DNS server

We also offer a fast DNS server of our own. Setting it up takes a few moments.

  1. Just click Enable DNS on the DNS tab in Site Details.
  2. Login to your domain provider's website.
  3. Go to the Name Servers configuration for your domain.
  4. Enter name servers the name servers provided by CloudCannon on the DNS tab.

Templating

Having the same header/footer across multiple pages can be a hassle when you want to make an update to your template. That's why CloudCannon has an include feature. On any HTML page you can use the following syntax to include any text based file <!-- include another_file.html -->.

Here's an example of how you would have a static header / footer across multiple pages:
First create four files - header.html, footer.html, index.html and about.html.

In header.html put the following:

<html>
	<head></head>
	<body>
		<div id="navigation">
			<a href="index.html"> Home </a>
			<a href="about.html"> About </a>
		</div>

		<div id="content">

And in footer.html put the following:

		</div>
	</body>
</html>

Now we can include the header and footer into our other HTML pages. Let's start with the index.html page:

<!-- include header.html -->
<h1> My Home page </h1>
<!-- include footer.html -->

and the about page:

<!-- include header.html -->
<h1> About Page </h1>
<!-- include footer.html -->

You now have a site setup with templates.

Active Links

CloudCannon automatically adds a class of cc-active to any <a> tags which point to the current page. Using some CSS and the cc-active class you can highlight the active page.

Pretty URLs

Getting rid of those ugly .html extensions is easy on CloudCannon.

Let’s say you have an about.html page, create a folder called about, put the about.html file in the about folder then rename about.html to index.html. Now you can link to /about.

Custom 404 pages

If you want to display your own 404 page when someone mistypes a URL just add a page called 404.html to the root your website.

301 Redirects

When you change the structure of your website you might want the old URLs to redirect to new pages. To set up 301 redirect on CloudCannon, first create a file called 301.txt in the root of your website. In this file you'll specify the redirects using this syntax: 

old_url new_url

You can have wildcards by using * and external domains will work too. Here's an example file:

/old_file.html /new_file.html
/category/* /new_category
/page http://google.com

Form Submissions

We've made it super easy to have contact forms on your website. First create a normal form element, make the method post and the action point to the page you want the user to be redirected to after the form submission. Next add the input elements for your contact form, then finally add a hidden field with a name of cc_emails[] and a value of the email address to send the form submission to. That's it! Now you have a working contact form! Here's an example form: 

<form method="post" action="/success.html">
  <label>First Name</label><input type="text" name="first_name" />
  <label>Surname</label><input type="text" name="surname" />
  <label>Message</label><textarea name="message"></textarea>
  <input type="hidden" name="cc_emails[]" value="mike@cloudcannon.com" />
  <input type="hidden" name="cc_emails[]" value="george@cloudcannon.com" />

  <input type="submit" value="Send Message" />
</form>

Hiding files from your Client

Any file starting with an underscore (e.g. _header.html) will be hidden in the client editor. This is especially useful if you want to hide your templating files from the client.

Website Optimizations

We periodically perform optimizations on CloudCannon websites to make the websites load as fast as possible. All local website assets referenced in the HTML and CSS are rewritten to be served by our CDN. On production websites we minify the CSS files using clean-css and the Javascript using Uglifier.

Sharing websites

Clients

Giving your clients access takes just a few seconds. Add a class of editable to any elements you want to be editable in the HTML. Then go to the sharing options for the website and set a password. Now you can direct clients to go to http://yoursite.com/update and type in the password you've just set.

Developer

You can give access to other developers to update your site. Go into sharing options, click on the developers tab then enter the email address of the developer. Note only the site creator has the website in their Dropbox, other developers can only update the website through the editor in CloudCannon.
